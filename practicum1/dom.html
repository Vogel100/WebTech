Document Object Model

The DOM (Document Object Model) a standard developed by W3C. The DOM is a standard on how to access documents and their contents, and as such is separated in three parts: Core DOM, XML DOM and HTML DOM. This website is focusing on the HTML DOM.

The HTML DOM is both a model and a programming interface. It defines:

  • The HTML elements as objects.
  • The properties of all HTML elements
  • The methods to access HTML elements
  • The events for all HTML elements

The HTML DOM tells you how to access, manipulate, add or remove HTML elements.

A DOM looks like (figuur 1). It’s a treelike model of a HTML document, where every element in the document is a node/object on the tree, with the document itself being the root node. Objects in this tree may be called upon or changed by using methods on the objects.

The HTML page that this tree represents looks like this: 


				<html>
				<head>
					<title>My title”</title>
				</head>
					 <body>
						<a href=””>My link</a>
						<h1>My header</h1>
					</body>
				</html>

Elements in the tree can be accessed by a manner of means, most notably their id, class, or tag. Traditional tree traversal is also a possibility: nodes can access their children, siblings can access the next or previous sibling, all nodes (except the root) can access their parent.
